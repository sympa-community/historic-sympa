21. Reception mode


21.1 Message topics

A list can be configured to have message topics (different from topics for classifying mailing lists. Users can subscribe to these message topics in order to select reception mail : a message can have many topics and subscribers will receive only mails that are tagged by topics they are subscribed to. Messages can be tagged automatically, by their sender or by the list moderator.

21.1.1 Message topic definition in a list

Message topics are defined by list parameters. Foreach new message topic, create a new msg_topic paragraph that defines the name and the tile of the topic. For automatic tagging, you can defines keywords (See (20.4.11, page ). To define on which part of the message Sympa looks for keywords, you have to define msg_topic_key_apply_on list parameter (See 20.4.12, page ). Tagging a message can be optional or it can be required. It is indicate by the msg_topic_tagging list parameter (See (20.4.13,page ).

21.1.2 Subscribing to message topic for list subscribers

This functionnality is only available in reception mode ``normal''. Subscribers can select message topic to receive messages tagged with this topic. To receive not tagged mail, you can subscribe to the ``other''. Message topics of a subscriber are stored Sympa database (subscriber_table,field ``topics_subscriber'').

21.1.3 Message tagging

First of all, if some msg_topic.keywords are defined, Sympa tries to tag messages automatically. For manual tagging by sender or list moderator, on Web interface, Sympa uses scenarios : if the resulted action is ``editorkey'' (for example, scenario send.editorkey) and send.privatekey), the list moderator will asked to tag the message. If the resulted action is ``request_auth'' (for example, send.privatekey), the message sender will be asked to tag the message. There are some available variables to customize tagging (topic, topic-sender, topic-editor, topic-auto, topic-needed).

In case of message tagging required and if there is no tagging, Sympa will ask to the list moderator. (See (13, page )

Tagging a message will create a topic information file in the /usr/local/sympa-gb/spool/topic/ spool. It is named by the listname and the Message-ID. For message distribution, a field ``X-Sympa-Topic' is added to the message in order to use filters on it.
