25. Using Sympa with LDAP

LDAP is a client-server protocol for accessing a directory service. Sympa provide various features based on access to one or more LDAP directories :

  • <#4129#>authentication using LDAP directory instead of sympa internal storage of password<#4129#>
    see ~#auth-conf#4130>, page~#auth-conf#4131>

  • <#4132#>named filters used in authorization scenario condition<#4132#>
    see ~#named-filters#4133>, page~#named-filters#4134>

  • <#5907#>LDAP extraction of list subscribers (see ~#par-user-data-source#4135>)<#5907#>

  • <#4136#>LDAP extraction of list owners or editors<#4136#>
    see ~#data-inclusion-file#4137>, page~#data-inclusion-file#4138>

  • <#4139#>mail aliases stored in LDAP<#4139#>
    see ~#ldap-aliases#4140>, page~#ldap-aliases#4141>
