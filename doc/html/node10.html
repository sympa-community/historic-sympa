Organization

Here is a snapshot of what Sympa looks like once it has settled down on your system. This also illustrates the Sympa philosophy, I guess. Almost all configuration files can be defined for a particular list, for a virtual robot or for the whole site.

  • /usr/local/sympa-ff/
    The root directory of Sympa. You will find almost everything related to Sympa under this directory, except logs and main configuration files.

  • /usr/local/sympa-ff//bin
    This directory contains the binaries, including the CGI. It also contains the default authorization scenarios, templates and configuration files as in the distribution. /usr/local/sympa-ff//bin may be completely overwritten by the make install so you must not customize templates and authorization scenarios under /usr/local/sympa-ff//bin.

  • /usr/local/sympa-ff//bin/etc
    Here Sympa stores the default versions of what it will otherwise find in /usr/local/sympa-ff//etc (task models, authorization scenarios, templates and configuration files, recognized S/Mime certificates, families).

  • /usr/local/sympa-ff//etc
    This is your site's configuration directory. Consult /usr/local/sympa-ff//bin/etc when drawing up your own.

  • /usr/local/sympa-ff//etc/create_list_templates/
    List templates (suggested at list creation time).

  • /usr/local/sympa-ff//etc/scenari/
    This directory will contain your authorization scenarios. If you don't know what the hell an authorization scenario is, refer to 13,page . Those authorization scenarios are default scenarios but you may look at /usr/local/sympa-ff//etc/my.domain.org/scenari/ for default scenarios of my.domain.org virtual robot and /usr/local/sympa-ff//expl/mylist/scenari for scenarios specific to a particular list

  • /usr/local/sympa-ff//etc/data_sources/
    This directory will contain your .incl files (see 17.7, page ). For the moment it only deals with files requiered by paragraphs owner_include and editor_include in the config file.

  • /usr/local/sympa-ff//etc/list_task_models/
    This directory will store your own list task models (see 16.8, page ).

  • /usr/local/sympa-ff//etc/global_task_models/
    Contains global task models of yours (see 16.8, page ).

  • /usr/local/sympa-ff//etc/web_tt2/ (used to be /usr/local/sympa-ff//etc/wws_templates/)
    The web interface (WWSympa) is composed of template HTML files parsed by the CGI program. Templates can also be defined for a particular list in /usr/local/sympa-ff//expl/mylist/web_tt2/ or in /usr/local/sympa-ff//etc/my.domain.org/web_tt2/

  • /usr/local/sympa-ff//etc/mail_tt2/ (used to be /usr/local/sympa-ff//etc/templates/)
    Some of the mail robot's replies are defined by templates (welcome.tt2 for SUBSCRIBE). You can overload these template files in the individual list directories or for each virtual robot, but these are the defaults.

  • /usr/local/sympa-ff//etc/families/
    Contains family directories of yours (see 18, page ). Families directories can also be created in /usr/local/sympa-ff//etc/my.domain.org/families/

  • /usr/local/sympa-ff//etc/my.domain.org
    The directory to define the virtual robot my.domain.orgdedicated to managment of all lists of this domain (list description of my.domain.orgare stored in /usr/local/sympa-ff//expl/my.domain.org). Those directories for virtual robots have the same structure as /usr/local/sympa-ff//etc which is the configuration dir of the default robot.

  • /usr/local/sympa-ff//expl
    Sympa's working directory.

  • /usr/local/sympa-ff//expl/mylist
    The list directory (refer to , page ). Lists stored in this directory belong to the default robot as defined in sympa.conf file, but a list can be stored in /usr/local/sympa-ff//expl/my.domain.org/mylist directory and it is managed by my.domain.orgvirtual robot.

  • /usr/local/sympa-ff//expl/X509-user-certs
    The directory where Sympa stores all user's certificates

  • /usr/local/sympa-ff//locale
    Internationalization directory. It contains message catalogues in the GNU .po format.

  • /usr/local/sympa-ff//spool
    Sympa uses 9 different spools (see 2.4, page ).

  • /usr/local/sympa-ff//src/
    Sympa sources.
