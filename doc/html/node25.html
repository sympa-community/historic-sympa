24. Using Sympa with LDAP

LDAP is a client-server protocol for accessing a directory service. Sympa provide various features based on access to one or more LDAP directories :

  • <#4038#>authentication using LDAP directory instead of sympa internal storage of password<#4038#>
    see ~#auth-conf#4039>, page~#auth-conf#4040>

  • <#4041#>named filters used in authorization scenario condition<#4041#>
    see ~#named-filters#4042>, page~#named-filters#4043>

  • <#5648#>LDAP extraction of list subscribers (see ~#par-user-data-source#4044>)<#5648#>

  • <#4045#>LDAP extraction of list owners or editors<#4045#>
    see ~#data-inclusion-file#4046>, page~#data-inclusion-file#4047>

  • <#4048#>mail aliases stored in LDAP<#4048#>
    see ~#ldap-aliases#4049>, page~#ldap-aliases#4050>
